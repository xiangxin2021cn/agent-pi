# v0.10.4 - Agent Pi Workspace Output Update

## Features

- **Workspace-first formal outputs** - sessions can promote process files into a formal `Agent Pi Outputs` directory under the active working folder, with source labels that distinguish formal outputs, attachments, data, plans, downloads, notes, and long responses.
- **Project-grouped conversations** - the session panel groups conversations by working folder and folds branch/sub-agent conversations under their main conversation, making the main working window easier to find.
- **Controlled model handoff** - idle conversations can switch models and LLM connections with a confirmation flow, preserving recent conversation context as carryover memory for the next runtime.

## Improvements

- **In-app file previews** - Markdown, text, code, JSON, Office, and spreadsheet files now open through bounded in-app previews instead of always falling through to the operating system.
- **Large attachment memory protection** - path-backed attachments keep large Excel/Office files out of renderer memory and use bounded server-side previews or metadata when foreground conversion would be unsafe.
- **Agent Pi branding refresh** - the desktop app icon, topbar, sidebar glass treatment, active navigation states, and sidebar brand panel now use the Agent Pi visual identity.
- **Cleaner shutdown** - app quit now awaits session runtime cleanup, helping stop Pi/Claude subprocesses, MCP pool servers, watchers, and messaging workers instead of leaving resident processes behind.

## Bug Fixes

- Fixed relative process files like `response.md` being passed to Windows Shell and resolving under the install directory.
- Fixed Markdown preview failures that could produce a blank fullscreen preview for real non-empty files.
- Hardened Windows packaging against transient `bun.exe`, `claude.exe`, and `rg.exe` copy/extraction file-lock failures.
- Added renderer-side tool-result clamping to reduce UI freezes from very large tool outputs.

## Breaking Changes

- None.
