# v1.1.2 - Task Contract Goal Loop

Agent Pi 1.1.2 upgrades Goal Loop from a generic completion checker into a task-contract driven review loop.

## Highlights

- Added a persisted task contract for work sessions. The contract captures the original request, follow-up requests, task type, deliverables, hard requirements, evidence requirements, output formats, acceptance criteria, forbidden shortcuts, and the bound working directory.
- Goal reviewer prompts now treat the task contract as binding, so review passes should reject scope reduction, skipped granularity, omitted hard constraints, and missing deliverables.
- Automatic improvement prompts now carry the full task contract into the next pass, reducing the risk that the second or third round fixes only one small issue while forgetting the original user intent.
- Document tasks now include a Document Plan for title, audience, tone, length, sections, tables, charts, citations, delivery formats, and readability enhancements.
- Visual enhancements such as charts, embedded HTML blocks, flow diagrams, and visual summaries are constrained to verified source data or explicit user input. If the data is unavailable, the agent must mark the visualization basis as unavailable instead of inventing it.
- Added a visible UI layer for the document-work engine: the input box now shows a document enhancement badge for likely document tasks, Goal mode labels are clearer, and the session info panel shows a compact Document Plan status line.
- Existing goal sessions without a task contract are upgraded lazily from their current objective before merging follow-up work instructions.
- Added conservative detection for obvious scope reduction such as replacing a requested work product with a brief outline, placeholder, or deferred follow-up.
