# v1.1.0 - Project gbrain Memory Backend

## Release Theme

Agent Pi v1.1 introduces the first project-scoped advanced memory backend. The goal is to give long-running document work a stronger knowledge boundary: the selected working directory owns its own project memory, and same-directory sessions can reuse prior verified outputs without leaking context across unrelated projects.

## Highlights

- Added project gbrain configuration in Workspace Settings with `local_pglite`, `local_postgres`, and remote MCP backend modes.
- Added local PostgreSQL + pgvector support for project gbrain, including a derived database per project working directory.
- Bound gbrain runtime to `GBRAIN_HOME`, `GBRAIN_NAMESPACE`, and `GBRAIN_SOURCE` for every project session.
- Added automatic gbrain source registration for the selected working directory before imports and initialization.
- Added background project gbrain initialization when a session binds a working directory, so local PostgreSQL project databases are prepared without blocking chat startup.
- Added post-import gbrain maintenance for project memory: link extraction, facts reconciliation, and stale embedding refresh are now attempted and recorded in the project manifest.
- Added compact right-panel project memory status so users can see whether gbrain refresh, links, facts, and embeddings are ready or pending.
- Improved Windows gbrain command execution so paths containing spaces or Chinese characters are passed as real arguments rather than shell-split text.
- Treated gbrain doctor embedding/resolver warnings as non-blocking when the database connection, pgvector extension, and schema version are healthy.
- Kept project gbrain out of workspace default source slugs so it is injected only when a session has a selected working directory.

## Local Test Setup

- Verified PostgreSQL 18 on port `5433`.
- Verified pgvector `0.8.2`.
- Verified gbrain CLI `0.42.53.0`.
- Uses the configured PostgreSQL URL as a service/database-name template, then derives a project database such as `agent_pi_gbrain_<project-hash>`.
- Registers each project source inside its own project database under a deterministic `project-<hash>` namespace.

## Notes

- v1.1 still treats gbrain as an advanced project memory backend, not a global shared memory source.
- Cross-project recall must be handled later through explicit company or industry knowledge sources.
- Embedding providers are optional at this stage. Without embeddings, Agent Pi still prepares and imports project memory text, while vector recall quality depends on later provider configuration.
- If the local gbrain embedding provider is missing credentials, Agent Pi keeps the text import successful and marks embeddings as pending instead of blocking the goal loop.
