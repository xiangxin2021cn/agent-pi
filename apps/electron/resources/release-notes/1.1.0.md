# v1.1.0 - Goal Loop and Project Memory Lite

## Release Theme

Agent Pi v1.1 focuses on zero-config document-work reliability. It keeps the Codex-style Goal Loop improvements, project working-directory isolation, formal output visibility, preview/export/edit workflow, and Project Memory Lite. External memory backends are not part of this release.

## Highlights

- Strengthened Goal Loop for long document tasks: output existence, output folder, file preview, source evidence, requested deliverables, and document quality are checked before the task is considered complete.
- Added compact document expert reporting for structure, evidence, numbers, specification, risk, and total score.
- Writes formal output review reports beside generated deliverables under `_reviews/*.review.md`.
- Keeps Project Memory Lite zero-config: each working directory owns `.agent-pi/brain`, with append-only `entries.jsonl`, `facts.jsonl`, `citations.jsonl`, source indexes, artifact events, and output review indexes.
- Loads bounded same-directory Project Memory Lite entries into new sessions so later conversations can reuse trusted project context without scanning unrelated folders.
- Locks project-bound sessions to their selected working directory after work begins, reducing cross-project memory pollution.
- Improves the right-side information panel so formal outputs remain visible and previewable instead of being buried by Goal Loop audit details.
- Keeps Markdown preview, rendered editing, save, download, PDF export, and DOCX export in the document preview flow.

## Notes

- This release deliberately avoids requiring users to configure databases, vector stores, API keys, or external project-memory services.
- Company or industry knowledge should remain an explicit user-enabled source. It is not silently merged into project-local memory.
