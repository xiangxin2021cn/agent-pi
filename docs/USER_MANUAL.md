# Agent π 用户操作手册

适用版本：Agent π Windows 桌面版，面向当前 GitHub Releases 发布通道。  
最后更新：2026-06-18。

本手册面向日常使用者，不要求理解代码。它解释 Agent π 的工作方式、主要界面、常用操作、文件和成果管理、模型切换、数据源、技能、自动化以及常见问题处理。

## 1. Agent π 是什么

Agent π 是一个桌面智能体工作台。它不是单纯的聊天窗口，而是把“项目工作目录、会话、分支智能体、模型、文件产物、正式输出和外部数据源”组织在一起，适合长期项目分析、招投标资料研究、工程文件处理、代码仓库改造和多模型协同。

它的核心设计目标有三点：

- 让主会话容易找回：会话默认按项目/工作目录组织，分支智能体折叠在主会话下。
- 让文件产物可追踪：附件、过程数据、下载文件和正式成果在右侧信息面板中分类展示。
- 让正式成果回到项目目录：设置工作目录后，正式输出会写入该目录下的 `Agent Pi Outputs`，而不是只留在应用缓存里。

## 2. 安装、启动和更新

1. 打开 GitHub 最新发布页：[Agent π Releases](https://github.com/xiangxin2021cn/agent-pi/releases/latest)。
2. 下载 `Agent-Pi-x64.exe`。
3. 双击安装程序并按提示完成安装。
4. 启动后，应用会使用同一个 GitHub Releases 通道检查自动更新。

如果自动更新异常，可以重新下载最新版 `Agent-Pi-x64.exe` 覆盖安装。覆盖安装通常不会删除工作区、会话记录或配置。

## 3. 核心概念

| 概念 | 含义 | 使用建议 |
| --- | --- | --- |
| Workspace | 应用内的工作空间，保存会话、数据源、技能、标签、状态、设置等。 | 可以按公司、项目群或用途划分。 |
| 工作目录 | 智能体真正读写项目文件的本地目录。 | 一个投标项目、代码仓库或资料文件夹应设置为工作目录。 |
| 会话 | 一次任务或一条对话线。 | 把主会话作为任务控制台，不要把所有子问题都开成平级会话。 |
| 分支/子智能体会话 | 智能体为并行研究、拆分任务或继续任务产生的派生会话。 | 在会话列表中折叠查看，用主会话汇总结论。 |
| 模型连接 | AI 提供商连接和具体模型，例如 DeepSeek、Qwen、Claude 兼容接口、视觉模型等。 | 在空闲会话中切换；执行中先停止再切换。 |
| 数据源 | API、MCP、本地文件夹等外部能力。 | 对重复使用的系统集成，应配置为数据源。 |
| 技能 | 特定任务的操作说明和工作流。 | 对投标分析、代码审查、文档生成等重复任务使用技能。 |
| 正式输出 | 被认为是最终成果的文件。 | 优先保存在工作目录的 `Agent Pi Outputs` 下。 |
| 会话目录 | 应用内部为每个会话保存附件、计划、数据和缓存的目录。 | 适合追溯过程资料，不建议作为正式交付位置。 |

## 4. 记忆和上下文如何工作

Agent π 的“记忆”主要来自当前会话和项目上下文，而不是一个无限制的全局记忆库。

- 同一会话会保存完整对话记录、任务状态、模型设置、工作目录和会话文件。
- 继续同一会话时，智能体会尽量沿用之前的对话上下文、决策、路径、限制和未完成事项。
- 新会话不会天然继承旧会话的全部聊天内容，但会共享同一 Workspace 的设置、数据源、技能、标签和工作目录默认值。
- 设置工作目录后，系统会把该目录中的项目上下文文件纳入提示，例如 `AGENTS.md`、`CLAUDE.md` 等。
- 如果需要让新会话继承某个旧会话的结论，最稳妥的做法是引用旧会话产出的正式文件，或让智能体读取旧会话的输出文档。

模型切换不会删除应用层的会话历史。跨连接切换时，底层 provider runtime 会重建，因此适合在任务暂停后切换，而不是在模型正在输出时切换。

## 5. 主界面导览

### 5.1 左侧导航栏

左侧导航栏用于进入主要功能区：

- **新建会话**：创建新的任务会话。
- **所有会话**：查看全部会话，可按项目、日期、状态、未读分组。
- **状态**：积压、待办、待审查、完成、已取消等会话状态筛选。
- **已标记 / 已归档**：查看重点会话或已归档会话。
- **标签**：按自定义标签组织会话。
- **数据源**：管理 API、MCP、本地文件夹。
- **技能**：查看、添加、删除 Workspace 或项目技能。
- **自动化**：配置定时任务、事件触发和智能体自动化。
- **设置**：进入 AI、Workspace、权限、外观、输入、标签、消息、服务器等设置。
- **最新动态**：查看更新说明。

左下角显示 Agent π / Always π AI Studio 品牌面板。

### 5.2 会话列表

中间偏左的会话列表用于定位任务。

常用操作：

- 点击列表顶部的筛选/分组按钮，可以按 **项目、日期、状态、未读** 分组。
- 默认项目分组会把相同工作目录下的会话放在一起。
- 主会话旁边出现数字和箭头时，表示其下有分支/子会话，点击可展开或折叠。
- 会话标题右键或打开标题菜单，可以重命名、标记、归档、设置状态、添加标签或在新窗口打开。
- 搜索会话时，列表会按搜索结果展示匹配的会话。

建议把每个实际项目都设置清晰的工作目录，这样“所有会话”会更接近 Codex 风格的项目视图。

### 5.3 主对话区

主对话区包含：

- 顶部会话标题和标题菜单。
- 消息历史、任务进度、工具调用、文件卡片和长输出。
- 底部输入框。
- 输入框附近的执行模式、状态、附件、数据源、工作目录、模型选择器和发送/停止按钮。
- 右下或右侧的 **信息** 按钮，用于打开会话信息面板。

如果智能体正在执行，发送按钮会变为停止按钮。需要中断长任务时，先停止再修改模型、工作目录或关键设置。

### 5.4 右侧信息面板

右侧信息面板是理解当前会话状态的关键入口。它通常显示：

- 标题
- 进度和消息数量
- 当前模型和连接
- 工作目录
- 正式输出目录
- 会话目录
- 来源状态
- 会话文件树

会话文件树会显示：

- `Official Outputs`：正式成果。
- `attachments`：上传附件。
- `data`：结构化数据、表格转换结果等。
- `downloads`：从数据源下载的文件。
- `plans`：计划文件。
- `long_responses`：被压缩展示的长响应。

文件右侧会带来源标识，例如“正式”“附件”“数据”等。

## 6. 创建和管理 Workspace

### 6.1 创建 Workspace

首次启动或切换 Workspace 时，可以选择：

- 创建新的 Workspace。
- 打开已有本地文件夹作为 Workspace。
- 连接远程 Workspace。

如果是处理真实项目，建议 Workspace 名称使用项目群或业务域，例如 `Tender Analysis`、`Code Projects`、`South Africa Bids`。

### 6.2 设置默认工作目录

进入 **设置 -> Workspace -> 默认工作目录**：

1. 点击选择目录。
2. 选择真实项目文件夹，例如投标项目资料目录或代码仓库根目录。
3. 新建会话会默认继承该工作目录。

如果某个会话需要使用不同目录，可以在输入框底部的工作目录徽标中单独切换。

## 7. 新建会话和发送任务

### 7.1 新建普通会话

1. 点击左侧 **新建会话**。
2. 确认底部工作目录是否正确。
3. 选择模型和连接。
4. 根据风险选择执行模式。
5. 输入任务并发送。

建议第一条消息包含：

- 要完成的目标。
- 需要读取的文件或目录。
- 希望产出的文件格式。
- 是否需要先调研、先列计划、还是直接执行。
- 正式成果应保存到哪里。

示例：

```text
请先调研当前工作目录中的招标文件和 BOQ，梳理关键约束、工程范围和报价风险。
先不要写正式报告，先给出分析计划、需要读取的文件清单和预计产出结构。
```

### 7.2 使用工作目录

输入框底部的文件夹徽标显示当前工作目录。点击它可以：

- 选择最近使用的目录。
- 打开系统目录选择器。
- 清除工作目录，改用会话内部目录。

工作目录决定：

- 智能体默认从哪里读取项目文件。
- 正式输出默认写到哪里。
- 项目级技能从哪里加载。
- 预览白名单包含哪些项目文件。

### 7.3 上传附件

点击输入框左下角的回形针按钮，选择一个或多个文件。也可以把文件拖入窗口。

附件会进入当前会话目录的 `attachments`，并在右侧文件树中显示。大型 Excel/Office 文件会尽量走路径托管和有界预览，减少界面卡死。

如果提示“附件无法存储或发送”：

- 检查文件是否被 Excel、WPS 或其它程序独占打开。
- 检查路径是否过长或包含异常权限。
- 尽量把文件放入当前工作目录或其子目录。
- 关闭占用文件的程序后重试。

### 7.4 选择执行模式

输入框附近的模式按钮用于控制智能体自主程度。常见方式：

- 探索/计划类模式：适合先阅读、分析、列方案，不直接改文件。
- 执行模式：允许智能体实施写入、生成文件、运行命令等操作。

推荐流程：

1. 先让智能体探索和制定方案。
2. 审阅计划、范围和风险。
3. 再切到执行模式完成文件写入或代码修改。

Workspace 设置中可以配置 `Shift+Tab` 循环切换哪些模式。

### 7.5 设置会话状态

输入框附近的状态按钮可把会话标为：

- 积压
- 待办
- 待审查
- 完成
- 已取消

状态用于过滤和看板式管理。长周期项目建议在每次阶段性完成后更新状态，避免列表中混入已完成任务。

## 8. 模型和连接

### 8.1 添加模型连接

进入 **设置 -> AI -> 连接**：

1. 点击 **添加连接**。
2. 选择或填写提供商、Base URL、API Key 等。
3. 保存后点击 **验证连接**。
4. 设置默认模型、默认思考级别和 Workspace 覆盖。

Agent π 可以使用多个已配置模型，不应只绑定某一个视觉模型或文本模型。

### 8.2 在会话中切换模型

在输入框右下角点击模型名称，可以切换当前会话使用的模型。

规则：

- 会话空闲时可以切换。
- 会话正在输出或执行工具时，必须先停止当前响应。
- 同一连接内切换模型最轻量。
- 跨连接切换会保留会话历史，但会重建底层 provider runtime，并在下一次回复时携带当前上下文。

这适合以下场景：

- 文本模型不支持图片时，切换到支持视觉的模型。
- 当前模型长输出不稳定时，切换到另一个已配置模型继续。
- 先用低成本模型整理资料，再用高能力模型审稿或处理复杂推理。

### 8.3 视觉能力

对 Claude 兼容接口、OpenAI 兼容接口或自定义模型，模型选择器中可能需要手动启用“支持图片”。

上传图片前请确认：

- 当前模型支持视觉输入。
- 当前连接的服务端确实支持图片消息。
- 图片文件没有被外部程序锁定。

如果当前模型不支持图片，建议切换到 Workspace 中已经配置好的视觉模型，而不是另开无上下文新会话。

## 9. 数据源

数据源用于把外部系统接入智能体。左侧 **数据源** 下包含：

- **API**：HTTP API、OAuth API、应用集成等。
- **MCP**：Model Context Protocol 工具服务器。
- **本地文件夹**：把某个本地目录作为可检索或可访问来源。

常用操作：

1. 进入 **数据源**。
2. 点击添加按钮，选择 API、MCP 或本地文件夹。
3. 填写名称、连接信息和认证方式。
4. 保存并测试。
5. 在新会话或当前会话输入框的数据源按钮中启用它。

建议：

- 临时读取的项目文件，用工作目录和附件更直接。
- 经常复用的系统，例如 GitHub、企业 API、知识库、本地资料库，配置成数据源。
- MCP 服务器需要本机依赖时，先在终端确认它可以正常启动。

## 10. 技能

技能是给智能体的专业工作说明。它适合把稳定流程固化下来，例如：

- 投标文件分析。
- BOQ / Excel 提取。
- 可研报告生成。
- 代码审查。
- 文档翻译。
- 发布检查。

技能来源分三层，优先级从高到低：

1. 项目技能：`<工作目录>\.agents\skills\...`
2. Workspace 技能：当前 Workspace 内配置的技能。
3. 全局技能：用户目录下的全局技能。

项目技能优先级最高，适合某个项目的专用规则。使用时可以在消息中明确提到技能名称，也可以从技能列表查看可用技能。

## 11. 自动化

自动化用于让 Agent π 在特定条件下自动运行任务。左侧进入 **自动化** 后，可以管理：

- 定时任务。
- 事件触发任务。
- 智能体自动任务。
- Webhook 或提示动作。

自动化页面常用操作：

- 添加自动化。
- 启用或停用。
- 复制自动化。
- 删除自动化。
- 运行测试。
- 查看最近活动。
- 打开自动化生成的会话。

建议先在普通会话中验证提示词和模型设置，再把稳定流程迁移为自动化。

## 12. 文件、预览和正式输出

### 12.1 文件在哪里

Agent π 会把文件分为两类：

- 过程资料：保存在会话目录下，例如附件、计划、数据转换结果、下载文件、长响应。
- 正式成果：保存到正式输出目录。

如果会话设置了工作目录，正式输出目录为：

```text
<工作目录>\Agent Pi Outputs\<会话ID>\
```

如果没有设置工作目录，正式输出会落到应用内部会话目录的 `outputs`。

### 12.2 查看正式输出

在右侧信息面板的 **会话文件** 区域：

- 点击 **正式输出**，会在 Explorer 中打开正式输出目录。
- 展开 `Official Outputs`，可以看到正式成果文件。
- 点击文件会尝试在应用内预览。
- 右键文件可打开、在 Explorer 中显示或执行其它文件操作。

### 12.3 把过程文件提升为正式成果

如果智能体先把文件生成在 `attachments`、`data` 或会话目录下，可以在右侧文件树中：

1. 找到对应文件。
2. 右键文件。
3. 选择 **提升为正式成果**。
4. 文件会复制或移动到正式输出目录，并显示“正式”标识。

这适合把阶段性报告、数据提取结果、表格转换结果提升为最终交付物。

### 12.4 应用内预览支持

右侧文件树单击或双击文件时，Agent π 会优先应用内预览。常见支持类型包括：

- Markdown、TXT、代码文件、JSON、YAML。
- PDF。
- PNG、JPG、JPEG、GIF、WEBP、SVG 等图片。
- HTML。
- Excel、CSV 和结构化数据的有界预览。
- 部分 Office 文档的文本或转换预览。

目录不会在应用内预览，点击目录会打开系统文件管理器。

### 12.5 文件预览的安全边界

Agent π 不是任意外部文件浏览器。为了避免应用被聊天内容诱导打开任意路径，文件预览只允许以下范围：

- 当前会话目录。
- 当前会话的工作目录。
- 当前会话的正式输出目录。

因此，如果预览页提示：

```text
Access denied: file path is outside allowed directories
```

通常说明该文件不在当前会话允许目录内。处理方式：

- 确认会话工作目录设置正确。
- 从右侧信息面板中的文件树点击文件，而不是点击聊天中不可信的绝对路径。
- 把要预览的文件放入工作目录或正式输出目录。
- 如果是旧版本生成的路径，升级到最新版后重新打开会话。

### 12.6 预览空白怎么办

如果文件真实存在但预览为空：

- 先点击右侧 **在 Explorer 中查看**，确认文件大小不是 0。
- 对 Markdown 或文本文件，确认编码不是异常二进制内容。
- 对 PDF，确认不是加密、损坏或扫描空白页。
- 对 Excel，确认工作簿没有被其它程序锁定。
- 对超大文件，先让智能体生成摘要、CSV 或 Markdown 版本，再预览轻量结果。

## 13. 会话组织最佳实践

### 13.1 主会话作为控制台

复杂任务建议采用：

- 一个主会话负责目标、计划、分派和最终汇总。
- 多个分支智能体负责不同文件、不同章节或不同研究方向。
- 主会话只保留关键结论和正式输出入口。

这样在“所有会话”里可以折叠分支，快速找到主窗口。

### 13.2 命名会话

会话标题应直接反映任务，而不是保留第一句话：

- `N3 招标文件综合分析`
- `BOQ 价格表结构提取`
- `Book 2 合同数据风险检查`
- `README 与用户手册更新`

点击顶部标题或会话列表的菜单可以重命名。

### 13.3 使用标签和状态

标签适合横向分类：

- 项目名。
- 专业方向。
- 优先级。
- 截止日期。
- 是否需要复核。

状态适合任务流转。比如先设为“待办”，执行完成后改为“待审查”，确认后改为“完成”。

## 14. 设置说明

进入左侧 **设置** 后，主要页面如下：

| 设置页 | 用途 |
| --- | --- |
| AI | 管理连接、默认模型、思考级别、缓存和 Workspace 覆盖。 |
| Workspace | 修改名称、图标、默认工作目录、默认数据源、本地 MCP、模式切换。 |
| 权限 | 查看探索模式默认规则和 Workspace 自定义规则。 |
| 外观 | 设置主题、语言、字体、连接图标、工具图标。 |
| 输入 | 设置发送快捷键、拼写检查和输入行为。 |
| 标签 | 管理标签层级和自动应用规则。 |
| Messaging | 连接 Telegram、WhatsApp、飞书 / Lark。 |
| 服务器 | 配置远程访问、端口、令牌和 TLS。 |

### 14.1 输入快捷键

在 **设置 -> 输入** 中可以选择：

- Enter 发送，Shift+Enter 换行。
- Ctrl+Enter 发送，Enter 换行。
- macOS 风格的 Cmd+Enter 发送。

Windows 用户通常选择 Enter 或 Ctrl+Enter。

### 14.2 权限规则

在 **设置 -> 权限** 中可以查看探索模式允许的命令和规则。探索模式适合：

- 读取文件。
- 搜索代码。
- 分析目录结构。
- 生成计划。

执行模式适合：

- 写文件。
- 移动或生成成果。
- 安装依赖。
- 运行构建、测试或打包命令。

## 15. 推荐工作流

### 15.1 招投标资料分析

1. 创建或选择项目 Workspace。
2. 设置工作目录为投标项目资料目录。
3. 新建主会话，命名为项目分析总控。
4. 上传或引用招标文件、BOQ、Excel、Addendum、技术规范。
5. 使用探索模式让智能体先列文件清单、分析计划和风险点。
6. 需要并行时，让主会话拆出分支智能体分别分析不同 Book、章节或表格。
7. 回到主会话汇总结论。
8. 要求把正式报告写入 `Agent Pi Outputs`。
9. 在右侧面板检查 `Official Outputs`，预览并在 Explorer 中查看。
10. 把会话状态改为“待审查”或“完成”。

### 15.2 大型 Excel / BOQ 处理

1. 尽量把 Excel 放在工作目录内。
2. 上传附件或直接让智能体从工作目录读取。
3. 要求先识别工作簿、工作表、有效区域、表头和关键字段。
4. 对超大表格，要求分块提取、生成 JSON/CSV/Markdown 中间结果。
5. 在右侧 `data` 中查看结构化结果。
6. 结果稳定后，右键提升为正式成果或要求智能体写入正式输出目录。

### 15.3 代码仓库改造

1. 用代码仓库根目录作为工作目录。
2. 新建会话，先要求智能体阅读 README、package 配置、关键模块和测试。
3. 在探索模式下确认方案。
4. 切换到执行模式实施修改。
5. 要求运行必要的 typecheck、lint、test 或 build。
6. 让智能体总结修改文件、验证结果和剩余风险。

### 15.4 多模型协同

1. 用成本较低或速度较快的模型做初步阅读和整理。
2. 如果需要图片、扫描件或界面截图，切换到已配置的视觉模型。
3. 如果需要复杂审稿或推理，切换到更强模型。
4. 每次跨连接切换前，确保会话已经停止输出。
5. 切换后第一条消息说明“请基于当前会话上下文继续”。

## 16. 常见问题

### 16.1 程序未响应或任务卡住

处理顺序：

1. 点击停止按钮。
2. 等待几秒让子进程清理。
3. 如果模型接口长时间无响应，切换到另一个已配置模型。
4. 检查 API Key、Base URL、模型名和网络。
5. 避免在模型正在输出时频繁切换模型或发送新任务。

### 16.2 退出后仍有驻留进程

新版已加强 runtime、子进程、MCP pool、watcher 和消息 worker 的退出清理。如果仍遇到驻留：

- 确认安装的是最新版。
- 先停止所有正在执行的会话再退出。
- 检查是否有外部 MCP 服务器或本地工具长时间运行。

### 16.3 模型切换不了

如果提示“请先停止当前响应，再切换模型”，说明当前会话正在执行。先停止或等待任务完成。  
如果切换后报错，通常是连接配置、模型名、接口兼容性或视觉能力配置问题。

### 16.4 图片上传后模型看不到

检查：

- 当前模型是否支持视觉。
- 自定义模型是否在模型选择器中启用了图片支持。
- API 服务端是否真正支持多模态消息。
- 图片是否损坏或文件格式不受支持。

### 16.5 聊天中的路径点击后找不到文件

旧会话或模型输出中可能包含过期路径，尤其是 `response.md`、安装目录下的临时文件或被移动的过程文件。优先使用右侧信息面板的文件树，它展示的是当前会话真实可访问的文件。

### 16.6 正式输出目录里没有文件

可能原因：

- 会话没有设置工作目录。
- 智能体只在消息中展示内容，没有写入文件。
- 文件还在 `data`、`attachments` 或会话目录中，没有提升为正式成果。

处理方式：

- 明确要求“请将最终成果写入正式输出目录”。
- 在右侧文件树中右键过程文件，选择“提升为正式成果”。
- 打开 `Official Outputs` 检查。

### 16.7 预览被拒绝访问

说明文件不在当前会话允许的预览目录内。把文件放到工作目录或正式输出目录，或从右侧文件树重新打开。

### 16.8 附件上传失败

检查：

- 文件是否被 Excel/WPS/Word 占用。
- 文件是否在网络盘、同步盘或权限受限目录。
- 文件路径是否过长。
- 文件是否过大且当前模型/接口不适合直接上传。

对大型 Excel，优先放在工作目录内，让智能体按路径读取或分块处理。

## 18. 使用原则

- 真实项目先设置工作目录，再开始会话。
- 主会话负责统筹，分支会话负责并行研究。
- 探索模式先调研，执行模式再改文件。
- 过程资料可以保留，但正式成果要进入 `Agent Pi Outputs`。
- 使用右侧信息面板找文件，不要依赖聊天中随口出现的绝对路径。
- 模型切换要在会话空闲时做，视觉任务要选择支持图片的模型。
- 长周期项目用状态、标签和项目分组维护秩序。
